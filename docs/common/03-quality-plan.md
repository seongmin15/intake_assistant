## 2. 보안 요구사항

### 2.1 보안 항목

| 카테고리 | 요구사항 | 구현 접근 |
|---------|---------|----------|
| input_validation | 사용자 입력 텍스트의 길이 제한 및 sanitization | Pydantic 모델에서 max_length 제한, YAML injection 방지 |
| transport_security | 클러스터 내부 통신 보안 | k3s 내부 service mesh, 외부 노출 시 TLS |


### 2.6 수용한 보안 리스크

| 리스크 | 수용 사유 | 재검토 시점 |
|--------|----------|-----------|
| 인증 없이 API 노출 | 내부 도구 수준이며 민감 데이터를 저장하지 않음. k3s 클러스터 내부 접근만 허용. | 외부 사용자에게 공개하거나 과금 체계 도입 시 |

## 3. 에러 처리 및 실패 시나리오

### 핵심 플로우별 분석

#### intake_data.yaml 생성

- **정상 흐름**: 사용자 입력 → 질문 응답 → Sonnet 호출 → 유효한 YAML 생성 → 아키텍처 카드 표시

- **실패 시나리오**:

| 시나리오 | 가능성 | 영향 | 대응 |
|---------|--------|------|------|
| Anthropic API 호출 실패 (네트워크/rate limit) | low | high | 지수 백오프로 3회 재시도 후 사용자에게 '잠시 후 다시 시도해주세요' 표시 |
| 생성된 YAML이 SDwC validation 실패 | medium | low | validate-retry 루프로 자동 보정 (에러 메시지 + YAML을 Sonnet에 재전달, 최대 2회). 재시도 후에도 실패 시 에러 로그 기록 + 사용자에게 '다시 생성' 표시. |
| SDwC API 접속 불가 | low | high | YAML 생성까지는 정상 동작. ZIP 생성 단계에서만 에러 표시. YAML 텍스트 복사 옵션 제공. |

#### ZIP 프로젝트 다운로드

- **정상 흐름**: '이대로 진행' → SDwC /validate 통과 → SDwC /generate → ZIP 스트림 응답 → 브라우저 다운로드

- **실패 시나리오**:

| 시나리오 | 가능성 | 영향 | 대응 |
|---------|--------|------|------|
| SDwC /generate 타임아웃 | low | high | 30초 타임아웃 설정, 실패 시 재시도 안내 |

# intake-assistant


> 대화형 AI를 통해 SDwC용 intake_data.yaml을 자동 생성하는 서비스

---

## 1. 개요

비개발자가 자유 텍스트로 만들고 싶은 서비스를 설명하면, AI가 3~4개의 핵심 질문을 통해 의도를 파악하고 SDwC가 처리할 수 있는 intake_data.yaml을 자동 생성한다. 사용자는 기술 결정 없이 아키텍처 카드로 결과를 확인하고, ZIP 프로젝트를 받는다.


---

## 2. 문제 정의

### 2.1 핵심 문제

SDwC의 intake_data.yaml은 896줄 분량의 복잡한 YAML 템플릿으로, 비개발자가 직접 작성하기 어렵다

### 2.2 이 문제를 겪는 사람

아이디어는 있지만 기술 지식이 부족한 비개발자, 초기 프로토타이핑을 빠르게 시작하고 싶은 개발자

- **심각도**: high
- **빈도**: occasional

### 2.3 현재 해결 방법과 한계

SDwC 템플릿을 수동으로 복사해서 직접 채워야 하며, 기술 결정(프레임워크, DB, 아키텍처 패턴 등)을 모두 사용자가 내려야 한다

**문제점:**

- 896줄 YAML 템플릿의 필드를 하나하나 이해하고 채우는 데 높은 진입 장벽
- enum 값 오류, 빈 값 블록 등 validation 실패가 빈번 (PoC v1에서 17건 에러)
- 비개발자는 기술 스택, 아키텍처 패턴 등 기술 결정을 내릴 수 없음


---

## 3. 왜 지금인가

SDwC v1이 안정화되어 YAML→ZIP 변환이 검증되었으므로, 앞단 입력 경험 개선이 다음 단계

- **트리거**: SDwC v1.34 시뮬레이션에서 AI가 유효한 intake_data.yaml을 생성할 수 있음이 검증됨 (validation 0건 에러)


---

## 4. 핵심 가치 제안

- **핵심 가치**: 자유 텍스트 설명과 3~4개 질문만으로 유효한 intake_data.yaml을 자동 생성
- **차별점**: A-B-C 모델로 사용자에겐 의도만 묻고, 기술 결정은 AI가 추론하며, 기본값은 자동 채움


---

## 5. 대상 사용자

### 5.1 사용자 페르소나

#### 비개발자 아이디어 소유자 (핵심)

- **설명**: 서비스 아이디어는 있지만 기술 지식이 없어서 intake_data.yaml을 직접 작성할 수 없는 사람
- **핵심 목표**: 자유 텍스트로 아이디어를 설명하고 질문에 답하는 것만으로 프로젝트 스캐폴딩 ZIP을 받고 싶다
- **페인 포인트**:
  - 896줄 YAML 템플릿의 기술 용어를 이해할 수 없음
  - 프레임워크, DB, 아키텍처 패턴 등 기술 결정을 내릴 수 없음
- **기술 수준**: beginner
